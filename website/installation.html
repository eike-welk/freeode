Installation

Linux

Go ahead, just do it.

If you don't feel like compiling the program you can downoad a binary version. It was compiled on a Suse 9.3 system with gcc version 3.3.5.

  • Extract the executable from the archive. This results in a file named 'siml'
  • Copy the executable to a directory in your PATH. Usually: '/usr/local/bin'

Windows

No attempt has been made to port the compiler to Windows, since the development happens on Linux. In principle however, everything should work on Windows. The Python language and the necessary libraries ( Boost, SiPy) are all available for Windows. So you're on your own there.

If you have a C++ compiler for Windows and some time, you could probably port the compiler to Windows with minimum effort. The Python part needs no change, since Python promises to be platform independent. I would be very glad to receive patches and binaries.
